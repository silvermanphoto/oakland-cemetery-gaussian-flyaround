3. Prime Directive verdicts

  1. The card idled 56% of its life — three windows, one cause. Every idle hour traces to a monitoring failure: a monitor whose exit couldn't wake a sleeping agent, a finished stage with no self-firing successor, and a watcher that hung mid-check. The work was never slow — the watching was broken.
  2. Waste concentrated where verification was skipped; verification paid for itself where used. The unverified image tag cost $7.40. The driver gate cost $0.60 and saved a second all-night burn.
  3. Stacked unknowns taxed the recovery. New host + fresh build + unproven transfer tool at once made each failure ambiguous — roughly 1.5 hours of forensics the canary rule exists to prevent.
  4. What worked: the phone sentinel rang as designed; the $5 kill-floor was never threatened; the trainings were clean first-try runs; every model byte-verified to two machines before teardown.

4. Fixes now standing (each idle window produced one)

  • Idle #1 → monitors are local, harness-tracked, bounded pollers whose exit is the wake-up.
  • Idle #2 → finished stages fire their successors themselves.
  • Idle #3 → hard timeouts on every remote check inside a watcher; loud exit after consecutive failures; a frozen watcher log is itself an alarm. (Field manual updated the same hour.)
  • Program level → the meter-sentinel — independent of any session — remains the layer that cannot forget.

5. Actions this audit triggers for the 36-tile rebuild

  1. Never pay for the build again: the compiled trainer is salvaged off the pod before teardown, so a future pod goes from boot to training in minutes, not nine hours.
  2. Transfer recipe is settled: pre-staged archive + the relay that moved 12 GB in 13 minutes.
  3. Fleet watchers inherit the hardened pattern — hard timeouts, loud exits, lead backstop from launch.

With those in place, this same program re-run today costs ≈ $4 — the basis on which the 36-tile rebuild's rental budget should be modeled.
